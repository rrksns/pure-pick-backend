# Analyze Error

사용자가 제시한 에러 메시지를 분석하고, PurePick 프로젝트 맥락에서 원인을 파악한 후 해결책을 제시합니다.

## 사용법
```
/analyze-error
```

그 다음 에러 메시지를 붙여넣으면 됩니다.

---

## 에러 분석 프로세스

사용자의 에러 메시지를 받으면 다음 순서로 진행합니다:

1. **에러 분류**: 에러 타입 식별 (Django, Elasticsearch, Redis, MySQL, Docker 등)
2. **프로젝트 맥락 확인**: CLAUDE.md의 설정과 비교하여 프로젝트 특이사항 파악
3. **원인 분석**: 에러 스택 트레이스 분석 및 관련 코드 검색
4. **해결책 제시**: 단계별 해결 방법 제시 (빠른 픽스 → 근본 원인 해결)
5. **예방책**: 같은 에러가 재발하지 않도록 예방 방법 제시

## 분석 범위

### 자주 발생하는 에러들
- **MySQL 연결 에러**: Docker healthcheck 확인, 마이그레이션 상태 확인
- **Elasticsearch 연결 에러**: ES 구동 시간 부족, 문서 인덱싱 확인
- **Redis 연결 에러**: Redis 데이터베이스 번호 확인 (DB 1 사용)
- **Elasticsearch TypeErrors**: `elasticsearch<8.0` 버전 확인
- **캐시 문제**: 캐시 키 포맷 검증, TTL 확인
- **API 400/500 에러**: 요청 파라미터 검증, 직렬화 문제 확인

### 검사 항목
- 관련 Docker 컨테이너 상태
- 관련 로그 파일 내용
- 관련 소스 코드 (models.py, views.py, settings.py 등)
- 설정 파일 검증

## 출력 형식

```
## 🔍 에러 분류
[에러 타입 및 심각도]

## 📊 원인 분석
[관련 코드 파일 위치 + 문제점 설명]

## ✅ 해결책
1. [빠른 해결]
2. [근본 원인 해결]
3. [검증 방법]

## 🛡️ 예방책
[재발 방지 방법]
```
